Безпека

API-ключ Нової Пошти

Потрібен, щоб клієнти обирали відділення зі списку при оформленні замовлення.

Як отримати API-ключ Нової Пошти

  1. 1. Зареєструйте бізнес-кабінет
    Перейдіть на new.novaposhta.ua і увійдіть або створіть акаунт (підійде номер телефону, як у відділенні НП).
  2. 2. Відкрийте розділ для розробників
    Меню → Налаштування → Безпека → вкладка Для розробників.
    Або пряме посилання: new.novaposhta.ua/dashboard/settings/developers
  3. 3. Створіть ключ
    Натисніть «Створити ключ» → введіть назву (наприклад, Patriot Store) → підтвердіть SMS-кодом.
  4. 4. Скопіюйте ключ
    Ключ показується один раз — одразу скопіюйте його повністю (довгий рядок латиницею та цифрами).
  5. 5. Вставте в поле вище
    Натисніть «Зберегти ключ». Після цього на сторінці оформлення замовлення з'явиться пошук міст і відділень Нової Пошти.
    6. Важливо: ключ безкоштовний. Не публікуйте його у відкритому доступі.

База даних (Supabase)

Підключення для особистого кабінету клієнтів та збереження замовлень онлайн.

Не публікуйте цей ключ. Зберігається лише в браузері адміна.

Налаштування авторизації

Вимкнути «Email not confirmed»
Authentication → Providers → Email → вимкніть Confirm email → Save.
Для продакшену краще залишити увімкненим і підтверджувати пошту.
Google вхід
1. Authentication → Providers → Google → увімкнути
2. Створіть OAuth у Google Cloud Console
3. Redirect URI: https://ВАШ-ПРОЄКТ.supabase.co/auth/v1/callback
4. Вставте Client ID і Secret у Supabase
Вхід по телефону (SMS)
Authentication → Providers → Phone → увімкнути → підключити Twilio/MessageBird
Без SMS-провайдера працює лише вхід через email або Google.
Redirect URLs
Authentication → URL Configuration:
• Site URL: адреса сайту (напр. https://cartonkaGG.github.io/patriot/)
• Redirect URLs: додайте .../auth-callback.html

Як підключити Supabase

  1. 1. supabase.com → New project
  2. 2. SQL Editor → data/supabase-setup.sql → Run
  3. 3. Settings → API → URL + anon key + service role key
  4. 4. Налаштуйте авторизацію (блок вище)
  5. 5. Товари та фото зберігаються в таблиці products і Storage product-images
